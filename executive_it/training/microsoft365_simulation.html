Azure Portal
Azure Active Directory, Subscriptions, at Conditional Access

Ano ang Azure AD?

Ang Azure Active Directory (Azure AD) o ngayon ay tinatawag na Microsoft Entra ID ay ang cloud-based identity at access management service ng Microsoft. Ito ang nagma-manage ng user accounts, authentication, at authorization sa lahat ng Microsoft 365 services.

Paano gumawa ng Azure AD user:
1. Pumunta sa Azure Active Directory → Users → + New user
2. Piliin ang "Create new user"
3. Punan: Display name, User principal name (email format), password
4. I-set ang UsageLocation (kailangan bago mag-assign ng license)
5. I-click ang Create

PowerShell: New-AzureADUser -DisplayName "Juan" -UserPrincipalName "juan@kalyx.onmicrosoft.com" ...
Conditional Access — Nagtatatakda ng mga kondisyon bago payagan ang user na mag-login. Halimbawa: "Kung hindi compliant ang device, huwag payagan." O kaya: "Kung hindi galing sa trusted location, kailangan ng MFA." Ito ang Zero Trust security model ng Microsoft.
App Registrations — Dito nirerehistro ang mga third-party o custom na apps na gumagamit ng Azure AD para sa authentication. Ginagamit ito para sa OAuth 2.0 at OpenID Connect integrations.

Microsoft Intune
Endpoint Management, Device Compliance, at App Management

Ano ang Microsoft Intune?

Ang Microsoft Intune ay isang cloud-based Mobile Device Management (MDM) at Mobile Application Management (MAM) solution. Pinamamahalaan nito ang mga device ng employees — Windows, iOS, Android, macOS — at tinitiyak na sumusunod sila sa security policies ng organisasyon.

Paano mag-enroll ng device:
Windows: Settings → Accounts → Access work or school → Connect → mag-login sa work account
iOS: I-download ang Company Portal app → sign in → sundin ang steps
Android: I-download ang Intune Company Portal → sign in → i-activate ang Device Admin
Compliance Policy — Tinutukoy nito kung ano ang "compliant" na device. Halimbawa: minimum OS version, BitLocker encryption, PIN requirement, at hindi rooted/jailbroken.
Remote Wipe — Kapag nawala o nanakaw ang device, puwedeng i-wipe ang lahat ng data remotely.
App Deployment — Puwedeng mag-push ng apps sa mga managed devices nang hindi kailangang puntahan ng IT ang bawat computer.

Exchange Admin Center
Email management, mail flow, at policies

Ano ang Exchange Online?

Ang Exchange Online ay ang cloud-based email service ng Microsoft 365. Pinamamahalaan nito ang lahat ng mailboxes, distribution lists, shared mailboxes, at mail flow rules ng organisasyon.

Paano gumawa ng mailbox: Ang mailbox ay awtomatikong nalilikha kapag gumawa ka ng bagong user at nag-assign ng Exchange Online license.
Mail Flow Rules — Puwede kang gumawa ng rules tulad ng: "Kung may 'CONFIDENTIAL' sa subject, i-encrypt ang email."
Retention Policies — Tinutukoy kung gaano katagal ang itatago ng emails. Halimbawa: 7 taon para sa legal/finance emails.
Data Loss Prevention (DLP) — Pinipigilan ang pagpapadala ng sensitive na impormasyon sa labas ng organisasyon.

Users & Groups
Manage identities, roles, at group memberships

User management sa M365

Ang lahat ng user accounts ay nakaimbak sa Azure Active Directory. Ang bawat user ay may UPN (User Principal Name), roles, licenses, at group memberships.

Gumawa ng bagong user: I-click ang "+ Add user" button sa itaas para mag-fill ng form at gumawa ng actual na user account. Subukan mo!
Mga uri ng grupo:
• Security group — Para sa access control
• Microsoft 365 group — May sariling mailbox, Teams, SharePoint site
• Distribution list — Para lang sa email
• Dynamic group — Awtomatikong nagdadagdag ng members
MFA (Multi-Factor Authentication) — Nagdadagdag ng extra layer ng security. Kahit na ma-compromise ang password, kailangan pa rin ng second factor.
Admin roles (Principle of Least Privilege):
• Global Admin — Pinaka-makapangyarihan
• User Admin — Manage users at groups
• Helpdesk Admin — Reset passwords
• Security Admin — Security policies
• Exchange Admin — Email management

SharePoint Admin
Sites, storage, at sharing policies

Ano ang SharePoint Online?

Ang SharePoint Online ay isang collaboration platform para sa pag-iimbak, pag-share, at pamamahala ng mga dokumento at impormasyon.

Mga uri ng SharePoint site:
• Team site — Para sa isang team o project
• Communication site — Company-wide announcements
• Hub site — Nagtatali ng maraming sites
Sharing settings:
• Anyone — Anonymous links (risky!)
• New and existing guests — External sharing allowed
• Existing guests only — Existing external users lang
• Only people in your organization — Internal only (pinaka-secure)
SharePoint storage: Ang default ay 1 TB + 10 GB per license. Sa 300 licenses = 1 TB + 3 TB = 4 TB total.

Licenses

Paano gumagana ang licensing sa M365?

Ang bawat user ay kailangang may license para magamit ang mga Microsoft 365 services. Importante ang UsageLocation ng user bago mag-assign ng license.

Paano mag-assign ng license: I-click ang "Assign License" button, piliin ang user at ang product. Tandaan: dapat may UsageLocation ang user bago mag-assign ng license.
